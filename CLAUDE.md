# Project: Edge AI Optimization Benchmark (YOLOv11n on Raspberry Pi 5)

## Goal
Benchmark YOLOv11n object detection across model formats on this Raspberry
Pi 5 and produce a GitHub-ready write-up. Target CV line:
"Benchmarked YOLOv11n across PyTorch, ONNX, NCNN and INT8 TFLite on Raspberry
Pi 5, achieving Nx speedup with X% mAP retention."

The deliverable is the comparison (speed + accuracy + write-up), not a new model.
Use pretrained COCO weights (yolo11n.pt). Do NOT train anything.

## Hardware & environment constraints (hard rules)
- This machine is a Raspberry Pi 5 running Ubuntu, ARM64, limited RAM. Treat it
  as a constrained device at all times.
- NEVER install full TensorFlow on this Pi. TFLite INT8 export happens in
  Google Colab; only `tflite-runtime` may be installed here for inference.
- Work inside the Python venv at ~/edgeai (activate before any pip install).
- Prefer small, incremental runs. Long benchmarks can thermally throttle the
  Pi; keep cooldown pauses between formats and log CPU temperature.
- No paid services, no new hardware. Everything free.

## Phases
1. Baseline: run `benchmark.py` with only yolo11n.pt present. Record FPS.
2. Export ONNX + NCNN on the Pi via `export_models.py`. Re-run benchmark.
3. TFLite INT8: exported in Colab by the user
   (`YOLO("yolo11n.pt").export(format="tflite", int8=True)` there), file
   copied here as yolo11n_int8.tflite. Install `tflite-runtime` if needed,
   re-run benchmark.
4. Accuracy: run Ultralytics `val` on COCO128 for each format, record
   mAP50-95 into an accuracy table (accuracy.md). If COCO128 download is slow,
   run it once and reuse.
5. Write-up: README.md with methodology, results table (results.md),
   accuracy table, one bar chart (matplotlib, save as PNG, no GUI backend --
   use matplotlib.use("Agg")), and a short explanation of quantization and
   why edge deployment differs from cloud/Colab inference.

## Files
- benchmark.py       main timing script (already written; extend, don't rewrite)
- export_models.py   ONNX + NCNN export
- test.mp4           user-supplied 30s test video
- results.csv/.md    generated by benchmark.py

## Style rules for the write-up
- Honest numbers only. If INT8 accuracy drops a lot, report it as a finding.
- Plain prose, no marketing language.
- Every result must be reproducible from the scripts in this repo.
